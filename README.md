# github Actions Parte 6

**Parte 6 - Docker e Services no GitHub Actions**
Na sexta parte, o material foca em como usar containers Docker e services dentro dos workflows.

Tópicos abordados:

- Como rodar Docker containers diretamente nos jobs, permitindo criar ambientes de execução personalizados.

- Uso de services para subir serviços auxiliares (como banco de dados, cache, etc.) no workflow sem precisar configurar manualmente.

- Exemplos práticos de como conectar o job principal a um service (como um banco de dados PostgreSQL ou Redis rodando em container).

- Melhores práticas para isolar ambientes, manter os testes consistentes e aproveitar o poder do Docker no CI/CD.

Essa parte é essencial para construir pipelines robustos e escaláveis, simulando ambientes reais de produção nos testes.
